## 💾 Comment utiliser ce backlog

### Pour ajouter une idée
```markdown
### Article X : [Titre]
**Concept :** [Quel concept IA]
**Activité potential :** [Idée d'activité]
**Notes :** [Détails, pistes, anecdotes]
**Brouillon :** [Où le fichier, ou "À créer"]
**Cible publication :** [Date estimée]
```

### Pour passer en "rédaction"
1. Tester l'activité avec les enfants
2. Noter réactions exactes
3. Créer un brouillon : `content/articles/brouillon-mon-slug.mdx`
4. Remplir frontmatter + contenu
5. Metter à jour ce backlog : `🔴 Brouillons en cours`

### Pour publier
1. Finaliser le brouillon
2. Renommer : `mon-slug.mdx` (enlever "brouillon-")
3. Tester local : `npm run dev`
4. Vérifier sur `/blog` et `/`
5. Updater ce backlog : `✅ Publié`
6. Push et déployer
